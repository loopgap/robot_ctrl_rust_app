# Serial Debug - 串口调试

> 串口数据收发、十六进制显示与统计

## 功能概述

Serial Debug 页面提供串口数据收发功能和调试能力。

### 主要功能

| 功能 | 说明 |
|------|------|
| 十六进制发送 | 支持 HEX 格式发送 |
| 十六进制接收 | HEX 格式显示接收数据 |
| 发送历史 | 保存发送记录 |
| 帧间隔分包 | 超时自动分包 |
| 数据统计 | TX/RX 字节数、帧数统计 |

## 发送功能

### 发送模式

| 模式 | 说明 |
|------|------|
| 文本发送 | 普通文本字符串 |
| HEX 发送 | 十六进制数据 |
| 模板发送 | 使用报文模板发送 |

### 发送格式

```
# 普通文本
Hello World

# HEX 格式
48 65 6C 6C 6F 20 57 6F 72 6C 64
```

### 发送历史

最近 10 条发送记录，点击可快速重发。

## 接收功能

### 显示格式

| 格式 | 说明 |
|------|------|
| HEX | 十六进制显示 |
| ASCII | ASCII 字符显示 |
| 混合 | HEX + ASCII 对照 |

### HEX 显示示例

```
[10:23:45.123] RX: 48 65 6C 6C 6F 20 57 6F 72 6C 64
                  Hello World
```

### 接收过滤

| 过滤类型 | 说明 |
|----------|------|
| 关键词过滤 | 只显示包含关键词的帧 |
| 长度过滤 | 只显示特定长度的帧 |
| 方向过滤 | 只显示 TX 或 RX |

## 数据统计

### 发送统计

| 统计项 | 说明 |
|--------|------|
| 发送帧数 | 已发送帧总数 |
| 发送字节 | 已发送字节总数 |
| 发送错误 | 发送失败次数 |

### 接收统计

| 统计项 | 说明 |
|--------|------|
| 接收帧数 | 已接收帧总数 |
| 接收字节 | 已接收字节总数 |
| 接收错误 | 接收错误次数 |

### 实时速率

| 指标 | 说明 |
|------|------|
| TX 速率 | 当前发送速度 (bytes/s) |
| RX 速率 | 当前接收速度 (bytes/s) |

## 帧间隔分包

### 配置参数

| 参数 | 说明 |
|------|------|
| 帧间隔超时 | 超过此时间则认为新帧开始 |
| 最大帧长 | 单帧最大长度 |

### 工作原理

当接收到的数据间隔超过设定的帧间隔超时时，自动切分到新帧。

```
原始数据: AA BB CC DD EE FF GG HH
              ↓
帧间隔超时: AA BB CC DD | EE FF GG HH
              ↓
分包结果:   帧1         帧2
```

## 快捷操作

| 操作 | 快捷键 | 说明 |
|------|--------|------|
| 发送 | Ctrl+Enter | 发送输入数据 |
| 清除接收 | Ctrl+L | 清空接收缓冲区 |
| 清除统计 | Ctrl+Shift+C | 重置统计数据 |

## 使用示例

### 基本发送接收

1. 配置串口参数（端口、波特率）
2. 打开连接
3. 在发送框输入数据
4. 点击发送或按 Ctrl+Enter
5. 查看接收区显示

### HEX 数据发送

1. 选择 HEX 发送模式
2. 输入 HEX 字符串（空格分隔）
3. 点击发送
4. 观察接收数据

### 批量发送

1. 在发送历史中选择要重发的条目
2. 点击发送按钮
3. 快速重发历史数据

## 常见问题

### 发送失败

- 检查串口是否已连接
- 确认端口没有被其他程序占用
- 检查数据格式是否正确

### 接收乱码

- 确认波特率匹配
- 检查数据位、停止位、校验位设置
- 确认字符编码正确
