# CANopen 工具

> 标准 CAN / CAN FD / EtherCAT CoE 三协议统一工具

## 支持的协议

| 协议 | CAN ID 长度 | 最大数据长度 | 典型应用 |
|------|-------------|--------------|----------|
| 标准 CAN | 11-bit | 8 bytes | 工业自动化 |
| CAN FD | 11-bit / 29-bit | 64 bytes | 高速数据传输 |
| EtherCAT CoE | 11-bit | 8 bytes | 工业以太网 |

## CANopen 协议栈

```
┌─────────────────────────────────────────────────────┐
│                CANopen 协议栈                       │
├─────────────────────────────────────────────────────┤
│  应用层 (Application)                              │
│  ├─ NMT (Network Management)                      │
│  ├─ SDO (Service Data Object)                    │
│  └─ PDO (Process Data Object)                    │
├─────────────────────────────────────────────────────┤
│  对象字典 (Object Dictionary)                      │
│  ├─ 索引 0x1000-0x1FFF: 通信参数                  │
│  ├─ 索引 0x2000-0x5FFF: 制造商特定参数            │
│  └─ 索引 0x6000-0x9FFF: 标准设备参数              │
├─────────────────────────────────────────────────────┤
│  CAN 底层 (CAN Base)                               │
│  └─ CAN 2.0 / CAN FD                             │
└─────────────────────────────────────────────────────┘
```

## NMT (网络管理)

### NMT 状态机

```
                 ┌──────────┐
        Busoff  │          │
    ───────────► │  Pre-    │
                 │  Operational│◄────────────┐
    ┌───────────►│          │◄─┐            │
    │            └──────────┘  │            │
    │                ▲         │ Operational │
    │                │         │            │
    │            Initialize    └────────────┤
    │                │                    │
    │            ┌────┴────┐               │
    └───────────►│          │──────────────┘
      Power On   │    Stop  │    Reset
                 │  (Stopped)│
                 └──────────┘
```

### NMT 命令

| 命令 | 命令字 | 说明 |
|------|--------|------|
| 启动 | 0x01 | 进入 Operational 状态 |
| 停止 | 0x02 | 进入 Stopped 状态 |
| 进入 Pre-Operational | 0x80 | 进入 Pre-Operational 状态 |
| 重置节点 | 0x81 | 重置节点（软件重启）|
| 重置通信 | 0x82 | 重置通信参数 |

### NMT 帧格式

```
┌────────────┬────────────┐
│ CAN ID     │ Data (2 bytes) │
├────────────┼────────────┤
│ 0x000      │ [CS] [NodeID] │
└────────────┴────────────┘
```

- **CS**: Command Specifier (命令字)
- **NodeID**: 节点 ID (1-127)

## SDO (服务数据对象)

### 用于读写对象字典

### SDO 请求 (客户端 → 服务器)

```
┌────────────────────────────────────────┐
│ CAN ID: 0x600 + NodeID (请求)          │
├────────────────────────────────────────┤
│ [2字节索引] [1字节子索引]              │
│ [请求内容: 1-4 字节数据]              │
└────────────────────────────────────────┘
```

### SDO 响应 (服务器 → 客户端)

```
┌────────────────────────────────────────┐
│ CAN ID: 0x580 + NodeID (响应)          │
├────────────────────────────────────────┤
│ [2字节索引] [1字节子索引]              │
│ [响应内容: 1-4 字节数据]              │
└────────────────────────────────────────┘
```

### SDO 加速传输

| 字节数 | 命令字 | 说明 |
|--------|--------|------|
| 1 字节 | 0x2F | 1 字节数据 |
| 2 字节 | 0x2B | 2 字节数据 |
| 4 字节 | 0x23 | 4 字节数据 |

### 常用对象字典索引

| 索引 | 对象 | 说明 |
|------|------|------|
| 0x1000 | Device Type | 设备类型 |
| 0x1001 | Error Register | 错误寄存器 |
| 0x1003 | Error History | 错误历史 |
| 0x1005 | COB-ID SYNC | SYNC 消息 ID |
| 0x1006 | Communication Cycle | 通信周期 (μs) |
| 0x1010 | Store Parameters | 保存参数 |
| 0x1017 | Producer Heartbeat Time | 心跳时间 (ms) |
| 0x1018 | Identity Object | 设备标识 |
| 0x1800-0x1803 | TPDO1-4 | 发送 PDO 参数 |
| 0x1A00-0x1A03 | RPDO1-4 | 接收 PDO 参数 |

## PDO (过程数据对象)

### TPDO (发送 PDO)

周期性发送数据：

```
┌────────────────────────────────────────┐
│ CAN ID: TPDO COB-ID                    │
├────────────────────────────────────────┤
│ [数据内容: 1-8 字节]                   │
└────────────────────────────────────────┘
```

### RPDO (接收 PDO)

接收外部数据：

```
┌────────────────────────────────────────┐
│ CAN ID: RPDO COB-ID                    │
├────────────────────────────────────────┤
│ [数据内容: 1-8 字节]                   │
└────────────────────────────────────────┘
```

### PDO 传输类型

| 类型 | 说明 |
|------|------|
| 0 | 同步非循环 |
| 1-240 | 同步循环 (每 N 个 SYNC) |
| 254 | 异步制造商特定 |
| 255 | 异步设备特定 |

## 心跳 (Heartbeat)

### 心跳格式

```
┌────────────────────────────────────────┐
│ CAN ID: 0x700 + NodeID                  │
├────────────────────────────────────────┤
│ [NMT 状态: 1 字节]                     │
└────────────────────────────────────────┘
```

### NMT 状态值

| 状态 | 值 |
|------|-----|
| Initializing | 0x00 |
| Pre-Operational | 0x7F |
| Operational | 0x05 |
| Stopped | 0x04 |

## CAN FD 特有功能

### 扩展数据长度

| DLC | 数据长度 |
|-----|----------|
| 0-8 | 与标准 CAN 相同 |
| 9-15 | 扩展长度 (12-64 bytes) |

### 比特率切换

BRS (Bit Rate Switch) 位用于切换高速数据阶段：

- 仲裁阶段: 低速 (如 500k)
- 数据阶段: 高速 (如 2M)

## EtherCAT CoE

### CANopen over EtherCAT

基于 EtherCAT 传输 CANopen 服务：

### CoE Mailbox 格式

```
┌─────────────────────────────────────────────────┐
│ Ethernet Header                                  │
├─────────────────────────────────────────────────┤
│ EtherCAT Header (11 bytes)                      │
├─────────────────────────────────────────────────┤
│ CANopen Mailbox                                 │
│ ├─ Header (6 bytes)                            │
│ │   ├─ Length                                  │
│ │   ├─ Address                                 │
│ │   └─ Channel / Flags                        │
│ └─ Data (N bytes)                              │
│    ├─ CANopen Header                           │
│    │   ├─ Command / Function Code             │
│    │   └─ Node ID                             │
│    └─ SDO Request / Response                  │
└─────────────────────────────────────────────────┘
```

### CoE SDO

与标准 CANopen SDO 类似，但通过 EtherCAT 传输：

| 命令 | 说明 |
|------|------|
| 0x01 | Download SDO Request |
| 0x02 | Download SDO Response |
| 0x03 | Upload SDO Request |
| 0x04 | Upload SDO Response |

## 使用示例

### 连接配置

```powershell
# CAN 2.0
Protocol: CAN 2.0
Baudrate: 500k
Node ID: 1

# CAN FD
Protocol: CAN FD
Arbitration Baudrate: 500k
Data Baudrate: 2M
Node ID: 1
```

### NMT 操作

```
1. 选择节点 ID
2. 发送 NMT 命令 (启动/停止/重置)
3. 观察状态变化
```

### SDO 读写

```
# 读取设备类型 (索引 0x1000)
1. 发送 SDO Upload Request (索引 0x1000, 子索引 0x00)
2. 接收 SDO Upload Response
3. 解析设备类型值

# 写入心跳时间 (索引 0x1017)
1. 发送 SDO Download Request (索引 0x1017, 值=1000)
2. 接收确认响应
```

## 诊断与排障

### 常见错误

| 错误码 | 说明 | 解决方案 |
|--------|------|----------|
| 0x0503 0000 | 触发看门狗 | 检查心跳发送 |
| 0x0601 0000 | 对象只读 | 使用正确命令 |
| 0x0602 0000 | 对象不存在 | 检查索引/子索引 |
| 0x0800 0000 | 通用错误 | 检查数据长度 |

### 排障步骤

1. 确认物理连接正常 (CAN_H / CAN_L)
2. 验证波特率匹配
3. 检查终端电阻 (120Ω)
4. 确认节点 ID 不冲突
5. 使用示波器检查总线电平
